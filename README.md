# Idea Clarifier Skill：快速评估你的想法的可行性

最近大家都在 Vibe Coding，灵感来得特别快——「我要不要做个 XX？」的念头，一天能冒出三个。脑子里想法越堆越多，人也越来越焦虑：到底哪个值得做？哪个该趁早扔掉？

真正难的从来不是有想法，而是**判断这个想法到底值不值得做**。大多数人这时只会去做传统的「搜索式」检验——东搜一篇西搜一篇，越查越乱，最后还是没底。问题出在：需求没有先用**工程化的方式澄清**，你搜的根本不是关键问题。

技术能不能落地、市场有没有真需求、凭什么是你、为什么是现在……这些问题，专业的人会拿一整套商学院框架去拆。而这个 skill，把**澄清 → 取证 → 评估**整套体系打包好了——**5 分钟左右**跑完，把没用的、评分低的想法干脆扔掉，只留下真正优质的那几个。

**这个 skill，就是把那套专业框架，变成你随口就能用的东西。**

---

## 🏛 它的核心：一套商学院级别的评估体系

别看它问你的都是大白话，**底层站着的是三套被反复验证的硬核方法论**：

| 你听到的大白话 | 背后真正在跑的框架 |
|------|------|
| 「谁、何时、为搞定什么会用它」「没有它你怎么凑合」 | **JTBD 用户任务理论**——硅谷产品圈与投资人专用，一刀切开「真需求」和「伪需求」 |
| 「已经有人做吗？凭什么选你」「为什么是现在」 | **波特五力 + 市场时机分析**——MBA 竞争战略的基本盘，判断你有没有立足之地 |
| 「实现难度 / 最小可行版本 / 最容易翻车的点」 | **工程可行性评估**——技术团队判断「能不能做、会死在哪」的标准打法 |

这三套东西，本来是要花钱请顾问、或者读半年商业书才能内化的。

**而这个 skill 做的事，就是把它们压缩成几句你随口就能答的问题——你享受「说人话」的轻松，拿到的却是「商学院级评估」的结论。**

更关键的是：它**不和稀泥**。跑完不是给你一句正确的废话，而是一个**能复算、能反驳、敢下判断的 0-10 分**。

---

## 它到底干嘛的？

一句话：**把「我有个想法……」这种说不清的念头，几分钟内变成一份敢下结论的可行性报告，外加一个 0-10 分。**

它不替你做产品，它帮你在动手之前先冷静地问一句：**这事儿，现在做，靠谱吗？**

---

## 它怎么帮你（主打一个快）

- **🗣 不让你填表**：你只管把想法甩出来，AI 先帮你把「想解决什么、谁会用、长什么样」填好，你只需点头或纠正。只有真正影响结论的关键问题，它才反过来问你（最多 2 个）。
- **🔍 联网取证，不靠瞎编**：竞品、市场、政策、风口这些判断，它会去全网搜索取证；动态事实优先看近一年的最新信息，稳定事实使用权威来源并标注日期——尤其平台最新封禁规则这种，模型记忆根本给不准。
- **⚖️ 技术 + 商业两层快速体检**：技术能不能落地、最容易翻车的点在哪；商业有没有真实痛点、凭什么是你、靠什么赚钱。
- **🎯 敢给分、敢说「先别做」**：最后给一个 0-10 分，并**列出加扣分明细**（比如：5 + 1(回报清晰) − 3(需求太低频) = 3），谁都能复算。
  - 7-10 分：✅ 值得做
  - 4-6 分：⚠️ 需调整（点名最弱的 1-2 项）
  - 0-3 分：❌ 暂缓（说清硬伤 + 给 2-3 个调整方向）

---

## 适合谁

- 点子很多、但不确定哪个值得投入时间的人
- 想做产品/内容/小程序，但对技术或商业心里没底的小白
- 跟 AI 聊了半天还是下不了决心的人

---

## 怎么用

直接对它说，比如：

> 「锐评一下我的这个想法：我想做一个帮自由职业者自动生成报价单的小工具」

它会自己跑完 澄清 → 联网核查 → 打分 全流程，给你一份《需求澄清报告》。觉得哪里假设错了，告诉它，它会**重算分数**给你看变化。

---

> 一句话总结：**别再用直觉赌方向了。让一套商学院级的方法论帮你过一遍——5 分钟左右，省下你 3 周的弯路。**
